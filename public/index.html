Designed around what the team sees first

A restrained product shell instead of a loud marketing wrapper.

The redesign focuses on the highest-visibility surfaces: homepage, login, and daily workspace chrome. The result is intentionally quieter, more structured, and closer to the kind of support software teams already trust.

What changed

Less demo theater. More software confidence.

  • Reduced noisy brand treatments and removed placeholder-y scaffold language from visible UI.
  • Shifted to softer neutrals, tighter typography, and clearer hierarchy across first-touch surfaces.
  • Made the shell feel closer to Intercom-style support tooling without rewriting backend behavior.
What stayed intact

Existing app behavior and routes.

  • Auth, helpdesk navigation, queue selection, triage controls, and API routes remain in place.
  • The landing page still points users into the same login and workspace flow.
  • The redesign is visual direction first, so future product work can layer in without redoing the shell again.
